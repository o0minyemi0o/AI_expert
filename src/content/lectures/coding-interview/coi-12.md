# 모의 인터뷰와 복기

## 왜 모의 인터뷰가 중요한가

알고리즘과 자료구조를 아무리 잘 알아도, 실전 면접에서 긴장하면 실력의 절반도 발휘하지 못합니다. 모의 인터뷰는 **실전과 동일한 환경에서 연습**하여 긴장감을 관리하고, 커뮤니케이션 능력을 훈련하는 유일한 방법입니다. 복기(回顧)는 같은 실수를 반복하지 않기 위한 체계적인 학습 전략입니다.

> **핵심 직관**: 모의 인터뷰 없이 실전에 나가는 것은 리허설 없이 무대에 오르는 것과 같습니다. 실력이 아니라 "실전 적응력"이 합격을 결정합니다.

## 1. 면접 형식별 특성

코딩 인터뷰는 형식에 따라 전략이 달라집니다.

| 형식 | 시간 | 특징 | 핵심 전략 |
|------|------|------|-----------|
| 전화 스크리닝 | 30-45분 | 공유 에디터, 음성만 | 말로 설명하는 연습 필수 |
| 온사이트 | 45-60분 | 화이트보드 or 노트북 | 손코딩 연습, 지우기 최소화 |
| 가상 면접 | 45-60분 | 화면 공유, 웹 IDE | IDE 단축키 숙지, 카메라 응시 |
| 테이크홈 | 2-4시간 | 자유 환경, 코드 품질 중시 | 테스트 코드, 문서화 포함 |
| 라이브 코딩 | 30-45분 | 페어 프로그래밍 스타일 | 협업 태도, 피드백 수용 |

## 2. 커뮤니케이션 프레임워크

면접관은 **결과보다 과정**을 봅니다. coi-01에서 다룬 5단계 프레임워크를 커뮤니케이션 관점에서 확장합니다.

```
면접 커뮤니케이션 흐름:

  문제 수령
    │
    ├─ "이 문제를 ~로 이해했는데, 맞나요?"        (확인)
    │
    ├─ "엣지 케이스로 ~가 있을 것 같은데요"        (능동적 질문)
    │
    ├─ "브루트포스로 O(n²)이 가능하고,             (접근법 제시)
    │    해시맵으로 O(n)까지 줄일 수 있습니다"
    │
    ├─ "이 부분은 투 포인터로 구현하겠습니다"       (구현 설명)
    │
    └─ "테스트 케이스 [1,2,3]으로 검증하면..."     (검증)
```

**피해야 할 커뮤니케이션 패턴:**

```
안 좋은 예                          좋은 예
──────────────────────────────────────────────────
5분간 침묵 후 코딩 시작            → 생각을 말로 설명하며 진행
"모르겠습니다"                     → "이 부분은 확실하지 않지만 ~로 접근해보겠습니다"
힌트 무시                          → "그 힌트가 ~ 방향을 의미하는 건가요?"
"당연히 이렇게 하면 됩니다"        → "이 접근의 장단점은 ~입니다"
```

> **핵심 직관**: 면접관이 힌트를 줄 때 그것을 무시하는 것은 가장 큰 감점 요인입니다. 힌트는 "이 방향으로 가라"는 신호이므로 반드시 활용하세요.

## 3. 시간 관리 전술

coi-01에서 전체 시간 배분을 다뤘다면, 여기서는 **위기 상황별 시간 전술**을 다룹니다.

| 상황 | 남은 시간 | 전술 |
|------|----------|------|
| 접근법을 못 찾겠을 때 | 10분 소모 시 | 브루트포스라도 구현 시작 |
| 구현 중 버그 발견 | 15분 미만 | 핵심 로직 우선, 헬퍼 함수 스킵 |
| 최적화를 못 했을 때 | 5분 미만 | 말로 최적화 아이디어 설명 |
| 예상보다 빨리 끝났을 때 | 10분 이상 | 팔로업 질문 대비, 코드 정리 |

```python
# 시간 관리 체크리스트 (45분 기준)
timeline = {
    "0-5분": "문제 이해, 엣지 케이스 질문",
    "5-15분": "접근법 설계, 복잡도 분석",
    "15-35분": "구현 (20분이 핵심!)",
    "35-45분": "테스트, 디버깅, 팔로업 대비",
}

# 위기 신호 감지
danger_signs = [
    "10분째 접근법을 못 정했다 → 브루트포스 선언",
    "20분째 첫 줄도 못 썼다 → 수도코드로 전환",
    "버그가 3개 이상이다 → 처음부터 다시 설계",
]
```

## 4. 모의 인터뷰 실전 시뮬레이션

실제 모의 인터뷰를 어떻게 진행하는지 시나리오로 봅니다.

```
모의 인터뷰 시나리오 (45분):

  면접관: "정렬된 두 배열의 중앙값을 구하세요. 전체 시간복잡도는 O(log(m+n))."

  [0-3분] 이해 단계
  "두 정렬된 배열이 주어지고... 합친 배열의 중앙값을 구하되
   실제로 합치지 않고 O(log) 시간에 구해야 한다는 거죠?"

  [3-10분] 설계 단계
  "브루트포스는 합치면 O(m+n)이지만, 로그 시간이 필요하니
   이분 탐색을 써야 합니다. 짧은 배열 기준으로 분할점을 찾으면..."

  [10-35분] 구현 단계
  (코드 작성하며 각 줄의 의도를 설명)

  [35-45분] 검증 단계
  "nums1=[1,3], nums2=[2]이면 중앙값은 2..."
  "빈 배열 케이스도 확인하겠습니다"
```

## 5. 복기(Post-mortem) 전략

모의 인터뷰 후 **체계적인 복기**가 없으면 같은 실수를 반복합니다.

```
복기 템플릿:

  1. 문제 정보
     - 유형: DP / 그래프 / ...
     - 난이도: Easy / Medium / Hard
     - 소요 시간: __분 (목표: __분)

  2. 과정 분석
     - 접근법 도달 시간: __분
     - 첫 정답 코드 완성: __분
     - 버그 수: __개
     - 힌트 사용: __회

  3. 무엇이 잘 되었나
     - (예: 엣지 케이스를 스스로 찾음)

  4. 무엇이 부족했나
     - (예: DP 상태 정의에 8분 소요)

  5. 액션 아이템
     - (예: 구간 DP 문제 5개 추가 풀기 → coi-07 복습)
```

> **핵심 직관**: 복기에서 가장 중요한 것은 "무엇을 몰랐는가"가 아니라 "왜 그 접근을 떠올리지 못했는가"입니다. 지식이 아닌 사고 과정의 갭을 찾아야 합니다.

## 6. 자가 평가 루브릭

면접관의 관점에서 자신을 객관적으로 평가하는 체크리스트입니다.

| 영역 | Strong | Acceptable | Weak |
|------|--------|-----------|------|
| 문제 이해 | 엣지 케이스까지 질문 | 기본 요구사항 파악 | 문제를 잘못 이해 |
| 접근법 | 최적해 직행 | 브루트포스 → 최적화 | 접근법 못 찾음 |
| 구현력 | 버그 없이 완성 | 1-2개 사소한 버그 | 핵심 로직 오류 |
| 커뮤니케이션 | 자연스럽게 설명 | 질문하면 설명 | 침묵 or 횡설수설 |
| 시간 관리 | 여유 있게 완료 | 시간 내 완료 | 시간 초과 |

## 7. 체계적 준비 로드맵

```
준비 단계별 체크리스트:

  [1주차] 기초 다지기
  ├─ coi-02~04: 배열, 문자열, 스택/큐 패턴 복습
  └─ 매일 Easy 2문제 풀기

  [2-3주차] 핵심 패턴
  ├─ coi-05~08: 트리/그래프, DP, 그리디, 이분탐색
  └─ 매일 Medium 2문제, 주 1회 모의 인터뷰

  [4주차] 실전 연습
  ├─ coi-09~11: 백트래킹, ML 코딩, 시스템 코딩
  └─ 매일 Medium/Hard 혼합, 주 2회 모의 인터뷰

  [면접 직전] 컨디션 조절
  ├─ 새 문제 X, 복기한 문제 재풀이
  └─ 시간 관리 연습, 충분한 수면
```

coi-01에서 시작한 코딩 인터뷰 전략은 이 강의의 모의 인터뷰와 복기로 완성됩니다. 알고리즘 실력(coi-02~09)과 실전 응용력(coi-10~11)을 갖춘 뒤, 체계적인 모의 인터뷰와 복기를 반복하는 것이 합격의 지름길입니다.

## 핵심 정리

- **모의 인터뷰**는 실전과 동일한 환경(시간 제한, 긴장감)에서 연습하여 적응력을 키우는 유일한 방법입니다
- 면접 중 **커뮤니케이션**이 코딩 실력만큼 중요하며, 생각 과정을 말로 설명하는 습관을 훈련해야 합니다
- **시간 위기** 시 브루트포스라도 구현하는 것이 아무것도 못 쓰는 것보다 월등히 좋은 결과를 냅니다
- 복기에서는 "무엇을 몰랐는가"보다 **"왜 그 접근을 떠올리지 못했는가"**를 분석해야 성장합니다
- 체계적 준비는 **기초(1주) → 핵심 패턴(2주) → 실전(1주) → 컨디션 조절** 순서로 진행합니다
